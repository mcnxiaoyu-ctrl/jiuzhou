# 九州修仙录 - 装备系统与强化设计

> 版本：v0.2  
> 日期：2026-02-07  
> 目标：记录装备成长（强化/精炼/镶嵌）已落地实现，统一规则、接口与回归口径。

## 1. 当前状态（已实现）

### 1.1 核心分层

- **物品定义（静态）**：`item_def`
- **物品实例（动态）**：`item_instance`
- **统一成长规则模块**：`server/src/services/equipmentGrowthRules.ts`
- **业务入口**：`server/src/services/inventoryService.ts`
- **路由入口**：`server/src/routes/inventoryRoutes.ts`

### 1.2 装备成长功能范围

- 强化：`0 ~ 15`，支持强化符与保护符
- 精炼：`0 ~ 10`，`+6` 起失败降 1 级
- 镶嵌：支持镶嵌、替换、卸下返还
- 展示一致性：背包、物品详情、市场统一走同一成长属性计算

## 2. 统一规则（equipmentGrowthRules）

实现文件：`server/src/services/equipmentGrowthRules.ts`

### 2.1 等级上限与倍率

- 强化上限：`ENHANCE_MAX_LEVEL = 15`
- 精炼上限：`REFINE_MAX_LEVEL = 10`
- 强化倍率：`1 + strengthen_level × 0.03`
- 精炼倍率：`1 + refine_level × 0.02`

最终基础属性显示值（装备）：

`display = 品质缩放(base_attrs) × 强化倍率 × 精炼倍率 + 宝石平值属性`

入口函数：`buildEquipmentDisplayBaseAttrs(...)`

### 2.2 强化成功率与失败惩罚

| 目标等级 | 成功率 |
|---:|---:|
| +1 ~ +5 | 100% |
| +6 | 80% |
| +7 | 70% |
| +8 | 60% |
| +9 | 50% |
| +10 | 40% |
| +11 | 35% |
| +12 | 30% |
| +13 | 25% |
| +14 | 20% |
| +15 | 15% |

失败等级：

- `+1 ~ +7`：失败不降级
- `+8 ~ +15`：失败降 1 级
- 若使用保护符（`protect_downgrade=true`）：失败不降级

### 2.3 精炼成功率与失败惩罚

| 目标等级 | 成功率 |
|---:|---:|
| +1 | 100% |
| +2 | 100% |
| +3 | 100% |
| +4 | 90% |
| +5 | 80% |
| +6 | 70% |
| +7 | 60% |
| +8 | 50% |
| +9 | 40% |
| +10 | 30% |

失败等级：

- `+1 ~ +5`：失败不降级
- `+6 ~ +10`：失败降 1 级

### 2.4 消耗公式

强化：`buildEnhanceCostPlan(itemLevel, targetLevel)`

- 材料：`target<=10 -> enhance-001`，否则 `enhance-002`
- 数量：1
- 银两：`max(50, floor((itemLevel + 5) * 20 * targetLevel))`
- 灵石：`max(0, floor(targetLevel / 5))`

精炼：`buildRefineCostPlan(itemLevel, targetLevel)`

- 材料：`enhance-002`
- 数量：`target>=8 ? 2 : 1`
- 银两：`max(100, floor((itemLevel + 8) * 35 * targetLevel))`
- 灵石：`max(0, floor((targetLevel + 1) / 3))`

## 3. 镶嵌规则

### 3.1 孔位上限

优先读取 `item_def.socket_max`；若 `<=0` 则按品质兜底：

- 黄：1
- 玄：2
- 地：3
- 天：4

### 3.2 孔位类型约束

- 支持 `item_def.gem_slot_types`（数组/按槽位对象）
- 宝石类型通过 `normalizeGemType(...)` 标准化
- 类型不匹配时拒绝镶嵌

### 3.3 `socketed_gems` 数据结构

`item_instance.socketed_gems` 为 JSON 数组，元素结构：

```json
[
  {
    "slot": 0,
    "itemDefId": "gem-001",
    "gemType": "attack",
    "effects": [
      { "attrKey": "wugong", "value": 3, "applyType": "flat" }
    ],
    "name": "攻击宝石·一级",
    "icon": "/assets/items/gem/gong_ji_bao_shi_1.png"
  }
]
```

说明：

- `effects` 当前仅 `applyType='flat'` 参与角色属性计算
- 替换镶嵌会先返还旧宝石到背包
- 卸下宝石会返还原宝石到背包

## 4. 路由接口（Inventory）

实现文件：`server/src/routes/inventoryRoutes.ts`

## 4.1 强化

`POST /api/inventory/enhance`

请求（兼容字段）：

```json
{
  "itemId": 123,
  "itemInstanceId": 123,
  "instanceId": 123,
  "enhanceToolItemId": 456,
  "protectToolItemId": 789
}
```

响应（核心字段）：

```json
{
  "success": true,
  "message": "强化成功",
  "data": {
    "strengthenLevel": 6,
    "targetLevel": 6,
    "successRatePermyriad": 9000,
    "roll": 1234,
    "usedMaterial": { "itemDefId": "enhance-001", "qty": 1 },
    "costs": { "silver": 1800, "spiritStones": 1 },
    "usedEnhanceToolItemDefId": "enhance-003",
    "usedProtectToolItemDefId": "enhance-005",
    "protectedDowngrade": false,
    "character": {}
  }
}
```

## 4.2 精炼

`POST /api/inventory/refine`

请求（兼容字段）：

```json
{ "itemId": 123 }
```

响应核心字段：`refineLevel / targetLevel / successRatePermyriad / roll / usedMaterial / costs / character`

## 4.3 镶嵌

`POST /api/inventory/socket`

请求（兼容字段）：

```json
{
  "itemId": 123,
  "gemItemId": 456,
  "gemItemInstanceId": 456,
  "gemInstanceId": 456,
  "slot": 0
}
```

响应核心字段：`socketedGems / socketMax / slot / gem / replacedGem? / character`

## 4.4 卸下宝石

`POST /api/inventory/socket/remove`

请求：

```json
{ "itemId": 123, "slot": 0 }
```

响应核心字段：`socketedGems / socketMax / removedGem / character`

## 4.5 典型失败语义

- 物品不存在 / 已锁定 / 交易中不可操作
- 材料不足 / 材料已锁定
- 银两不足 / 灵石不足
- 强化已达上限 / 精炼已达上限
- 该物品不可强化 / 不可精炼 / 不可镶嵌
- 该宝石不可镶嵌 / 类型与孔位不匹配
- 该孔位没有已镶嵌宝石

## 5. 展示一致性（已统一）

以下入口已统一使用 `buildEquipmentDisplayBaseAttrs(...)`：

- 背包接口：`GET /api/inventory/items`
- 物品详情：`server/src/services/itemService.ts#getItemInstance`
- 市场列表：`server/src/services/marketService.ts#getMarketListings / getMyMarketListings`

结果：背包详情、物品详情、拍卖行显示属性与角色实战结算口径一致。

## 6. E2E 验证脚本

脚本：`server/src/__e2e_enhance.ts`

覆盖点：

1. 强化：成功/失败分支、强化符消耗、保护符失败保级
2. 精炼：成功/失败分支、`+6` 起失败降级、上限失败
3. 镶嵌：首次镶嵌、指定槽位替换、卸下返还、空孔卸下失败、非法槽位失败
4. 状态限制：锁定失败、交易中失败
5. 资源限制：材料不足失败、货币不足失败
6. 穿戴态刷新：镶嵌/卸下后角色属性即时变化与回滚

输出：脚本最终打印结构化 JSON `summary`，并在关键断言失败时直接抛错退出。
